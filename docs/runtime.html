How it works

Your component source is evaluated as a JavaScript module. After customElements.define() registers the element, the rendering harness discovers it and uses @lit-labs/ssr to produce Declarative Shadow DOM.

In a Go application, this same flow happens inside the WASM module: component JS is provided alongside HTML, both are fed to the QuickJS runtime, and the rendered HTML comes back.
